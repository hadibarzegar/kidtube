<objective>
Wire worker.Enqueue into the CreateEpisode and RetryJob handlers so the YouTube-to-HLS ingestion pipeline actually executes at runtime. This closes the critical integration defect found in the v1.0 milestone audit where Job documents are created in MongoDB but never enqueued to the worker goroutine for processing.

Purpose: Without these two function calls, the entire video ingestion pipeline (yt-dlp download, FFmpeg transcode, HLS serving) is dead code that only runs on server restart via ResumeJobs. This fix makes jobs process immediately on creation and retry.
Output: Two modified Go handler files with worker.Enqueue calls, compiling cleanly.
</objective>

```go
        EpisodeID: episode.ID,

        Status:    models.JobStatusPending,
        CreatedAt: now,
```

```go
    res, err := database.Collection(db.CollJobs).InsertOne(ctx, job)

    worker.Enqueue(worker.JobRequest{
        JobID:     job.ID,
        EpisodeID: episode.ID,
        SourceURL: req.SourceURL,
        Source:    "youtube",
    })
    statusCode = http.StatusAccepted
}
```
