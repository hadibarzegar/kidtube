<objective>
Sequential ingestion worker that downloads video via yt-dlp and transcodes to multi-rendition HLS via FFmpeg.

Purpose: This is the core video pipeline. A single goroutine reads from a buffered channel (never concurrent ‚Äî avoids YouTube 429s), downloads the source video, transcodes to three HLS renditions (360p, 480p, 720p), and writes segments to the shared Docker volume served by nginx.

Output: Worker package with Start/Enqueue functions, FFmpeg multi-rendition transcoding, job status tracking in MongoDB.
</objective>

  <action>
    Create `backend/internal/worker/processor.go`:

    **Package structure:**
    ```go
    package worker

    type JobRequest struct {
        JobID     bson.ObjectID
        EpisodeID bson.ObjectID
        SourceURL string
    }
    ```

    **Exported API:**
    - `var jobQueue = make(chan JobRequest, 100)` ‚Äî buffered channel, never blocks HTTP handler
    - `func Start(ctx context.Context, db *mongo.Database, hlsRoot string)` ‚Äî launches a single goroutine that reads from jobQueue in a select loop. On `ctx.Done()`, stops processing.
    - `func Enqueue(req JobRequest)` ‚Äî sends to jobQueue channel (called from episode CREATE handler)

    **processJob internal function (called for each job):**

    1. **Set status to "downloading"**: Update job in MongoDB: `Status = JobStatusDownloading`, `StartedAt = now`, `UpdatedAt = now`

    2. **Download via yt-dlp:**
       - Output path: `filepath.Join(hlsRoot, req.EpisodeID.Hex(), "source.mp4")`
       - `os.MkdirAll(filepath.Dir(outputPath), 0755)`
       - Command: `exec.CommandContext(ctx, "yt-dlp", "--format", "bestvideo[ext=mp4]+bestaudio[ext=m4a]/best[ext=mp4]", "--merge-output-format", "mp4", "--sleep-requests", "5", "--no-playlist", "-o", outputPath, req.SourceURL)`
       - Capture stderr for error reporting
       - On failure: set status to "failed" with error message, return

    3. **Set status to "transcoding"**: Update job status in MongoDB

    4. **Transcode to HLS via FFmpeg:**
       - Output directory: `filepath.Join(hlsRoot, req.EpisodeID.Hex())`
       - Use `-force_key_frames "expr:gte(t,n_forced*6)"` instead of fixed `-g 180` (fps-agnostic, per RESEARCH.md recommendation)
       - Use `-sc_threshold 0` to disable scene-cut keyframes
       - FFmpeg command with three renditions:
         ```
         ffmpeg -i {inputPath}
           -filter_complex "[0:v]split=3[v1][v2][v3];[v1]scale=w=1280:h=720[v1out];[v2]scale=w=854:h=480[v2out];[v3]scale=w=640:h=360[v3out]"
           -map [v1out] -c:v:0 libx264 -b:v:0 2800k -maxrate:v:0 2996k -bufsize:v:0 4200k
           -map [v2out] -c:v:1 libx264 -b:v:1 1400k -maxrate:v:1 1498k -bufsize:v:1 2100k
           -map [v3out] -c:v:2 libx264 -b:v:2 800k -maxrate:v:2 856k -bufsize:v:2 1200k
           -map a:0 -c:a:0 aac -b:a:0 192k -ac 2
           -map a:0 -c:a:1 aac -b:a:1 128k -ac 2
           -map a:0 -c:a:2 aac -b:a:2 96k -ac 2
           -force_key_frames "expr:gte(t,n_forced*6)" -sc_threshold 0
           -f hls -hls_time 6 -hls_playlist_type vod
           -hls_flags independent_segments -hls_segment_type mpegts
           -hls_segment_filename {outDir}/%v/seg%03d.ts
           -master_pl_name master.m3u8
           -var_stream_map "v:0,a:0 v:1,a:1 v:2,a:2"
           {outDir}/%v/playlist.m3u8
         ```
       - Create subdirectories `0/`, `1/`, `2/` inside outDir before running FFmpeg
       - Capture stderr for error details
       - On failure: set status to "failed" with error, return

    5. **Set status to "ready"**: Update job: `Status = JobStatusReady`, `CompletedAt = now`

    6. **Update episode status**: Set episode `Status = "ready"` in `db.CollEpisodes`

    7. **Clean up**: `os.Remove(outputPath)` ‚Äî delete source.mp4 to save disk space

    **Helper function `updateJobStatus`:**
    ```go
    func updateJobStatus(ctx context.Context, db *mongo.Database, jobID bson.ObjectID, status models.JobStatus, errMsg string) {
        update := bson.D{
            {Key: "$set", Value: bson.D{
                {Key: "status", Value: status},
                {Key: "error", Value: errMsg},
                {Key: "updated_at", Value: time.Now()},
            }},
        }
        // Also set started_at on downloading, completed_at on ready
        db.Collection(db.CollJobs).UpdateOne(ctx, bson.D{{Key: "_id", Value: jobID}}, update)
    }
    ```

    **On retry (job status reset to pending by handler):** The worker picks up pending jobs on startup too. Add a `ResumeJobs` function that queries for pending jobs on startup and enqueues them. This handles both retries and server restarts with unfinished jobs.
