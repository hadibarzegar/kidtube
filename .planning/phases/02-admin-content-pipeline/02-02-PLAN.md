  <action>
    Create three handler files following the same dependency-injection pattern as HealthHandler (each function accepts `*mongo.Database` and returns `http.HandlerFunc`).

    **All three files share the same CRUD pattern:**
    - `List{Resource}(db) http.HandlerFunc` — `Find(ctx, bson.D{})` then `cursor.All(ctx, &slice)`, return JSON array. If slice is nil, return `[]` (never null JSON).
    - `Get{Resource}(db) http.HandlerFunc` — Extract ID from `chi.URLParam(r, "id")`, parse with `bson.ObjectIDFromHex(id)`, `FindOne` by `_id`, return JSON object or 404.
    - `Create{Resource}(db) http.HandlerFunc` — Decode JSON body, set `CreatedAt` and `UpdatedAt` to `time.Now()`, `InsertOne`, set `ID = res.InsertedID.(bson.ObjectID)`, return 201 + JSON.
    - `Update{Resource}(db) http.HandlerFunc` — Extract ID, decode JSON body, build `$set` document with `UpdatedAt: time.Now()`, `UpdateOne` by `_id`, return 200 + updated fields or 404 if `MatchedCount == 0`.
    - `Delete{Resource}(db) http.HandlerFunc` — Extract ID, `DeleteOne` by `_id`, return 204 or 404 if `DeletedCount == 0`.

    **channel.go specifics:**
    - Create/Update request body includes `name`, `description`, `thumbnail` (image URL string), `category_ids` (array of hex strings), `age_group_ids` (array of hex strings)
    - Parse `category_ids` and `age_group_ids` from hex strings to `[]bson.ObjectID` — return 400 if any ID is invalid
    - This satisfies CONT-05 (category and age group assignment to channels)

    **category.go specifics:**
    - Simple — only `name` field on create/update

    **age_group.go specifics:**
    - Fields: `name`, `min_age` (int), `max_age` (int) on create/update
    - Validate `min_age < max_age` and both >= 0 — return 400 if invalid

    **Important patterns (from STATE.md decisions):**
    - Always decode into typed Go structs (never bson.M)
    - Use `db.Collection(db.CollChannels)` etc. for collection name constants
    - Set `Content-Type: application/json` header on all responses
    - Use `chi.URLParam(r, "id")` for path parameters (chi router pattern)

    **Error response format:** `{"error": "message"}` with appropriate HTTP status code.
