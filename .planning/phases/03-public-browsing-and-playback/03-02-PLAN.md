<interfaces>
<!-- Site-api endpoints from Plan 03-01 -->

Site-api public endpoints (no auth):
- GET /channels — returns Channel[]; supports ?category_id=&age_group_id=
- GET /channels/{id} — returns Channel
- GET /episodes — returns Episode[] (status=ready only); supports ?channel_id=; sorted by order asc
- GET /episodes/{id} — returns Episode (status=ready only)
- GET /categories — returns Category[]; sorted by name asc
- GET /age-groups — returns AgeGroup[]; sorted by min_age asc
- GET /search?q= — returns {channels: Channel[], episodes: Episode[]}

Server-side: apiServerFetch('/channels') via SITE_API_INTERNAL_URL (Docker DNS)
Client-side: apiFetch('/channels') via nginx /api/site/ proxy

TypeScript types (define these inline or in a types file):
```typescript
interface Channel {
  id: string
  name: string
  description: string
  thumbnail: string
  category_ids: string[]
  age_group_ids: string[]
  created_at: string
  updated_at: string
}

interface Episode {
  id: string
  channel_id: string
  title: string
  description: string
  order: number
  subtitle_url: string
  status: string
  created_at: string
  updated_at: string
}

interface Category {
  id: string
  name: string
  created_at: string
  updated_at: string
}

interface AgeGroup {
  id: string
  name: string
  min_age: number
  max_age: number
  created_at: string
  updated_at: string
}
```

HLS path convention: `/hls/{episode_id}/master.m3u8` (used for thumbnail fallback if no thumbnail field on episodes)
