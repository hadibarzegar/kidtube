    1. Create `backend/internal/handler/site_channel.go`:
       - Package `handler`
       - Import: `encoding/json`, `net/http`, `db`, `models`, `bson`, `mongo`, `chi/v5`, `mongo/options`
       - `SiteListChannels(database *mongo.Database) http.HandlerFunc`:
         - Build a `bson.D{}` filter
         - If `?category_id=` query param present: parse to `bson.ObjectID`, add `{Key: "category_ids", Value: oid}` to filter (MongoDB matches array contains)
         - If `?age_group_id=` query param present: parse to `bson.ObjectID`, add `{Key: "age_group_ids", Value: oid}` to filter
         - Find with filter, sort by `created_at` descending: `options.Find().SetSort(bson.D{{Key: "created_at", Value: -1}})`
         - Return JSON array (empty array `[]` not null when no results — use `make([]models.Channel, 0)`)
       - `SiteGetChannel(database *mongo.Database) http.HandlerFunc`:
         - Get `{id}` from chi URL param, parse to ObjectID
         - FindOne by `_id`, return 404 if not found
         - Return JSON channel object

    2. Create `backend/internal/handler/site_episode.go`:
       - `SiteListEpisodes(database *mongo.Database) http.HandlerFunc`:
         - **CRITICAL**: Always include `{Key: "status", Value: "ready"}` in the filter — public endpoints must never expose unplayable episodes
         - If `?channel_id=` query param present: parse to ObjectID and add to filter
         - Sort by `order` ascending: `options.Find().SetSort(bson.D{{Key: "order", Value: 1}})`
         - Return JSON array (empty array not null)
       - `SiteGetEpisode(database *mongo.Database) http.HandlerFunc`:
         - Get `{id}` from chi URL param
         - FindOne with filter `bson.D{{Key: "_id", Value: oid}, {Key: "status", Value: "ready"}}` — only return ready episodes
         - Return 404 if not found or not ready

    3. Create `backend/internal/handler/site_category.go`:
       - `SiteListCategories(database *mongo.Database) http.HandlerFunc`:
         - Find all categories with no filter, sorted by `name` ascending
         - Return JSON array

    4. Create `backend/internal/handler/site_age_group.go`:
       - `SiteListAgeGroups(database *mongo.Database) http.HandlerFunc`:
         - Find all age groups with no filter, sorted by `min_age` ascending
         - Return JSON array

    5. Create `backend/internal/handler/site_search.go`:
       - Import: `encoding/json`, `net/http`, `regexp`, `strings`, `sync`, `db`, `models`, `bson`, `mongo`
       - `SiteSearch(database *mongo.Database) http.HandlerFunc`:
         - Read `q` query param; trim whitespace; if empty return `{"channels":[],"episodes":[]}`
         - Build case-insensitive regex: `bson.Regex{Pattern: regexp.QuoteMeta(q), Options: "i"}`
         - Run two queries in parallel using `sync.WaitGroup`:
           - Channels: filter on `name` field with regex
           - Episodes: filter on `title` field with regex AND `status: "ready"` (never return non-ready episodes in search)
         - Combine results into `map[string]any{"channels": channels, "episodes": episodes}`
         - Return JSON response
         - On either query error, log it and return what succeeded (graceful degradation)

       const SITE_API_INTERNAL_URL =
         process.env.SITE_API_INTERNAL_URL ?? 'http://localhost:8081'

       /**
        * Server-side fetch wrapper — calls site-api directly via Docker internal DNS.
        * Used in Server Components; no credentials needed (all public endpoints).
        */
       export function apiServerFetch(path: string, options?: RequestInit): Promise<Response> {
         return fetch(`${SITE_API_INTERNAL_URL}${path}`, {
           ...options,
           headers: {
             'Content-Type': 'application/json',
             ...(options?.headers ?? {}),
           },
         })
