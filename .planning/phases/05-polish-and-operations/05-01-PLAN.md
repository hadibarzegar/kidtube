  <name>Task 2: Create UploadEpisode multipart handler and wire route in admin-api</name>

**handler/upload.go** — Create a new file with the `UploadEpisode` handler. This handler:

1. Accepts `database *mongo.Database` and `hlsRoot string` parameters (same pattern as how worker.Start receives hlsRoot).
2. Caps request body with `http.MaxBytesReader(w, r.Body, 2*1024*1024*1024)` (2 GiB).
3. Calls `r.MultipartReader()` for streaming multipart parsing (NOT `r.ParseMultipartForm`).
4. Pre-allocates `episodeID := bson.NewObjectID()` before the multipart loop so the output directory is known.
5. Creates the output directory: `os.MkdirAll(filepath.Join(hlsRoot, episodeID.Hex()), 0755)`.
6. Iterates `mr.NextPart()` reading text fields first (channel_id, title, description, order, subtitle_url) using `io.ReadAll(io.LimitReader(part, N))` with small limits (256-4096 bytes per text field).
7. For the "file" part, streams with `io.Copy(dst, part)` directly to `{hlsRoot}/{episodeID.Hex()}/source.mp4`. Close dst explicitly after io.Copy (not defer — we need to detect write errors before responding).
8. After the multipart loop completes, validates: fileWritten must be true, channelID must be non-empty, title must be non-empty. On validation failure, clean up with `os.RemoveAll(outDir)` and return 400.
9. Parses channelID to `bson.ObjectID` via `bson.ObjectIDFromHex`.
10. Creates Episode document (same fields as CreateEpisode: ChannelID, Title, Description, Order, SubtitleURL, Status: "pending", timestamps) and inserts into episodes collection.
11. Creates Job document with `Source: "upload"`, `SourceURL: ""`, `Status: models.JobStatusPending`, and inserts into jobs collection.
12. Calls `worker.Enqueue(worker.JobRequest{JobID: job.ID, EpisodeID: episode.ID, Source: "upload"})`.
13. Returns 202 Accepted with the episode JSON.

**Error handling:** If io.Copy fails (including MaxBytesReader limit), clean up the partial output directory with `os.RemoveAll(outDir)` and return 413 if the error message contains "too large", otherwise 500.

**Import the worker package:** `"github.com/hadi/kidtube/internal/worker"` — same import used in episode.go if it were calling Enqueue.

Use the same JSON error response pattern as other handlers:
```go
w.Header().Set("Content-Type", "application/json")
w.WriteHeader(statusCode)
json.NewEncoder(w).Encode(map[string]string{"error": "message"})
```

  <name>Task 3: Configure nginx upload proxy location with increased body size and timeouts</name>
  <files>nginx/nginx.conf</files>
  <action>
Add a specific location block for the upload endpoint BEFORE the generic `/api/admin/` location block in nginx.conf. This location needs increased body size and timeouts to handle 2GB file uploads:

```nginx
# Admin API upload — large body size for video file uploads (up to 2GB)
location = /api/admin/episodes/upload {
    client_max_body_size 2100m;
    proxy_read_timeout 600s;
    proxy_send_timeout 600s;
    proxy_request_buffering off;

    proxy_pass http://admin_api/episodes/upload;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

Key configuration:
- `client_max_body_size 2100m` — allows up to ~2.1GB (slightly over 2GB to account for multipart overhead). Set per-location, NOT globally, to avoid creating a large-body attack surface on other endpoints.
- `proxy_read_timeout 600s` and `proxy_send_timeout 600s` — 10 minutes for slow connections uploading large files.
- `proxy_request_buffering off` — stream the upload directly to the backend without buffering in nginx temp files (important for 2GB files).
- Exact match (`= /api/admin/episodes/upload`) takes priority over prefix match (`/api/admin/`).
- `proxy_pass` URL includes the full path (`/episodes/upload`) since this is an exact match location, not a prefix strip.

Place this block immediately BEFORE the existing `location /api/admin/ {` block.

**Do NOT:**
- Set `client_max_body_size` globally or on the generic `/api/admin/` location.
- Remove or modify the existing `/api/admin/` location block — it stays unchanged for all other admin API requests.
