<task type="auto">
  <name>Task 1: Add map directive for per-extension HLS cache headers in nginx</name>
  <files>nginx/nginx.conf</files>
  <action>
Use the nginx `map` directive approach (recommended in research) to set per-extension cache headers without falling into the `add_header` inheritance trap.

**1. Add a `map` block** in the `http { }` block, BEFORE the `server { }` block:

```nginx
# Per-extension cache control for HLS files
map $uri $hls_cache_control {
    ~\.ts$    "public, max-age=31536000, immutable";
    ~\.m3u8$  "no-cache";
    default   "public, max-age=3600";
}
```

This maps:
- `.ts` segments -> immutable (content-addressed, never change after creation)
- `.m3u8` playlists -> no-cache (client must revalidate to get current playlist)
- Everything else -> 1 hour cache (safe fallback)

**2. Update the `/hls/` location block** to use the map variable instead of the hardcoded cache header:

Replace the existing line:
```nginx
add_header Cache-Control "public, max-age=3600";
```

With:
```nginx
add_header Cache-Control $hls_cache_control always;
```

Also add `always` to the existing CORS `add_header` directives so they apply on all response codes (including 206 partial content for range requests):
```nginx
add_header Access-Control-Allow-Origin * always;
add_header Access-Control-Allow-Methods 'GET, OPTIONS' always;
add_header Access-Control-Allow-Headers 'Range, Origin, Accept' always;
add_header Access-Control-Expose-Headers 'Content-Length, Content-Range' always;
```

The `always` parameter ensures headers are sent on ALL responses including 2xx, 3xx, 4xx — critical for CORS on 206 partial content responses used by Video.js for range-based HLS segment requests.

**Key advantage of the map approach:** All `add_header` directives stay in a single location block. No nested locations, no inheritance issues, no CORS header loss. The `$hls_cache_control` variable is evaluated per-request based on the URI before the location block executes.

**Do NOT:**
- Use nested `location ~ \.ts$ {}` blocks inside `/hls/` — this causes `add_header` inheritance loss and silently drops CORS headers.
- Remove the CORS headers — Video.js requires them for HLS playback.
- Use the `ngx_headers_more` module — it requires rebuilding the nginx Docker image.

Also remove the old comment `# Cache headers (Phase 5 will tune further)` — this IS Phase 5. Replace with a clear comment explaining the map-based approach.
